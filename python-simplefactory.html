简单工厂模式

我将连续写三个工厂模式，个人理解，这里先说简单工厂模式 Simple factory也就是通过一个工厂类的参数来决定产生的产品类, 也就是说从工厂的不同方法访问不同的需要被生产的类

python的例子

我这里搞了个例子

class Factory(object):
    def register(self, methodName, constructor, *args, **kargs):
        """注册产品"""
        _args = [constructor]
        _args.extend(args)
        # 给类添加属性，值就是相应间接调用类的实例
        setattr(self, methodName, apply(Functor, _args, kargs))

    def unregister(self, methodName):
        """取消注册"""
        delattr(self, methodName)

class Functor(object):
    def __init__(self, function, *args, **kargs):
        # 确定function是可以被调用的
        assert callable(function),
        self._function = function
        self._args = args
        self._kargs = kargs

    def __call__(self, *args, **kargs):
        """重载了括号运算符, 让工厂类的相关属性成为可调用的"""
        _args = list(self._args)
        _args.extend(args)
        _kargs = self._kargs.copy()
        _kargs.update(kargs)
        return apply(self._function, _args, _kargs)

看调用，每次注册一个属性，就可以调用这个属性返回当时设置属性的类的实例
